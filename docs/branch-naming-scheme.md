## Branch Naming Scheme

For branches, I use names that are clear and predictable.

### Format

```text
type/short-description
```

### Types

* `feat/`   new features
* `fix/`       bug fixes
* `refactor/`  code improvements
* `docs/`      documentation work
* `chore/`     maintenance or tooling

### Examples

```text
feat/inventory-system
fix/null-reference-input
refactor/resource-manager
docs/update-readme
chore/project-setup
```

### Naming Notes

I usually stick to these habits:

* keep everything lowercase
* use hyphens instead of spaces
* keep it short, but still descriptive
* avoid vague names like `stuff` or `update`
